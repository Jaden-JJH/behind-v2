## 🎯 7단계 개선 계획

### ✅ 1단계: Viewport 메타태그 및 메타데이터 수정
**예상 시간:** 5분
**우선순위:** 🔴 최우선
**영향도:** 매우 높음

**작업 내용:**
- `app/layout.tsx`에 viewport 설정 추가
- metadata title, description 수정

**변경 파일:**
- `app/layout.tsx`

**Before:**
```tsx
export const metadata: Metadata = {
  title: "Create Next App",
  description: "Generated by create next app",
};
```

**After:**
```tsx
export const metadata: Metadata = {
  title: "Behind - 모두의 뒷얘기 살롱",
  description: "지금 가장 뜨거운 이슈의 뒷이야기",
  viewport: {
    width: "device-width",
    initialScale: 1,
    maximumScale: 5,
    userScalable: true,
  },
};
```

**검증 방법:**
- 모바일 브라우저에서 확대/축소 없이 적절한 크기로 표시되는지 확인

---

### ✅ 2단계: 이슈 목록 이미지 크기 최적화
**예상 시간:** 30분
**우선순위:** 🔴 최우선
**영향도:** 높음

**작업 내용:**
- 이슈 목록 썸네일 크기 증가
- sm: breakpoint 추가로 단계적 크기 조정

**변경 파일:**
- `components/issues/IssuesListClient.tsx`

**Before:**
```tsx
<div className="relative w-32 h-24 md:w-48 md:h-32 ...">
```

**After:**
```tsx
<div className="relative w-40 h-32 sm:w-44 sm:h-36 md:w-48 md:h-32 ...">
```

**변경 사항:**
- 모바일: 128px×96px → 160px×128px (33% 증가)
- 중형 모바일: 176px×144px
- 태블릿 이상: 192px×128px

**검증 방법:**
- iPhone SE, iPhone 14 Pro, Galaxy S23에서 썸네일 크기 확인
- 이미지가 선명하게 보이는지 확인

---

### ✅ 3단계: sm: breakpoint 전반 도입 (padding/margin)
**예상 시간:** 2시간
**우선순위:** 🟠 높음
**영향도:** 높음

**작업 내용:**
- 주요 레이아웃 컴포넌트에 sm: breakpoint 추가
- 단계적 여백 조정 (px-3 → px-3 sm:px-4 md:px-6)

**변경 파일:**
- `app/page.tsx`
- `app/issues/page.tsx`
- `app/issues/[id]/page.tsx`
- `components/issues/IssuesListClient.tsx`
- `components/Header.tsx`
- `components/Footer.tsx`

**패턴:**
```tsx
// Before
className="px-3 md:px-4"

// After
className="px-3 sm:px-4 md:px-6"
```

**적용 대상:**
- Container padding: px, py
- 섹션 간격: space-y, gap
- 버튼 간격: gap

**검증 방법:**
- 576px, 640px, 768px, 1024px에서 레이아웃 확인
- 여백이 자연스럽게 증가하는지 확인

---

### ✅ 4단계: 텍스트 크기 반응형화
**예상 시간:** 2시간
**우선순위:** 🟠 높음
**영향도:** 높음

**작업 내용:**
- 제목, 본문 텍스트에 sm: breakpoint 추가
- 단계적 텍스트 크기 조정

**변경 파일:**
- `components/issues/IssuesListClient.tsx`
- `app/issues/[id]/page.tsx`
- `app/page.tsx`

**패턴:**
```tsx
// 제목 (h1)
className="text-2xl sm:text-3xl md:text-4xl"

// 제목 (h2)
className="text-xl sm:text-2xl md:text-3xl"

// 제목 (h3)
className="text-base sm:text-lg md:text-xl"

// 본문
className="text-sm sm:text-base"

// 작은 텍스트
className="text-xs sm:text-sm"
```

**적용 대상:**
- 이슈 제목
- 이슈 설명 (preview)
- 댓글 본문
- 카드 제목
- 버튼 텍스트

**검증 방법:**
- 모바일에서 텍스트 가독성 확인
- 확대 없이 읽을 수 있는지 확인

---

### ✅ 5단계: 댓글 영역 모바일 최적화
**예상 시간:** 1시간
**우선순위:** 🟠 높음
**영향도:** 중간

**작업 내용:**
- Textarea 높이 증가
- 버튼 크기 및 간격 조정
- padding 증가

**변경 파일:**
- `app/issues/[id]/page.tsx`

**Before:**
```tsx
<CardContent className="p-3 md:p-4">
  <Textarea className="min-h-[80px] md:min-h-[100px] mb-2 md:mb-3 ..." />
  <div className="flex items-center gap-1.5 md:gap-2">
```

**After:**
```tsx
<CardContent className="p-4 sm:p-5 md:p-6">
  <Textarea className="min-h-[100px] sm:min-h-[120px] md:min-h-[140px] mb-3 md:mb-4 ..." />
  <div className="flex items-center gap-2 sm:gap-3 md:gap-4">
```

**변경 사항:**
- Textarea 최소 높이: 80px → 100px (모바일)
- CardContent padding: 12px → 16px
- 버튼 간격: 6px → 8px

**검증 방법:**
- 모바일에서 댓글 입력 시 충분한 공간이 있는지 확인
- 터치로 버튼 클릭이 편한지 확인

---

### ✅ 6단계: Touch Target 크기 보장
**예상 시간:** 2시간
**우선순위:** 🟡 중간
**영향도:** 중간

**작업 내용:**
- 모든 버튼/링크 최소 44×44px 확보
- 아이콘 크기 조정
- 간격 증가

**변경 파일:**
- `app/issues/[id]/page.tsx`
- `components/issues/IssuesListClient.tsx`
- `components/Header.tsx`

**패턴:**
```tsx
// 작은 버튼
<Button size="sm" className="min-h-[44px] min-w-[44px]">

// 아이콘 크기
<ThumbsUp className="w-4 h-4 sm:w-5 sm:h-5" />
```

**적용 대상:**
- 투표 버튼 (좋아요/싫어요)
- 댓글 작성 버튼
- 신고 버튼
- 필터 버튼

**검증 방법:**
- 실제 모바일 기기에서 버튼 터치 테스트
- 인접 버튼 오터치 발생 여부 확인

---

### ✅ 7단계: 필터 UI 모바일 최적화
**예상 시간:** 2시간
**우선순위:** 🟡 중간
**영향도:** 중간

**작업 내용:**
- 카테고리 필터를 horizontal scroll로 변경
- 정렬 필터 개선

**변경 파일:**
- `components/issues/IssuesListClient.tsx`

**Before:**
```tsx
<div className="flex flex-wrap gap-2">
  {categories.map(cat => (...))}
</div>
```

**After:**
```tsx
<div className="overflow-x-auto -mx-3 px-3 sm:mx-0 sm:px-0">
  <div className="flex gap-2 min-w-max pb-2">
    {categories.map(cat => (...))}
  </div>
</div>
```

**변경 사항:**
- 모바일: horizontal scroll (wrap 방지)
- 태블릿 이상: wrap 허용
- 스크롤바 스타일링

**검증 방법:**
- 모바일에서 가로 스크롤이 자연스러운지 확인
- 모든 카테고리에 접근 가능한지 확인
